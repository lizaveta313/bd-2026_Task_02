# calc_data

Решение задачи обработки большого parquet-файла с тиковыми сделками Binance по одной торговой паре.

Программа читает parquet-файл и вычисляет:

- сумму `|VWAP_buy - VWAP_sell|` по календарным секундам;
- сумму `|VWAP_buy - VWAP_sell|` по календарным минутам.

`VWAP` считается по формуле:

```text
VWAP = sum(price * quantity) / sum(quantity)
```

Вывод строго пишется в `stdout`:

```text
VWAP_s=<число>
VWAP_m=<число>
```

Значения печатаются с точностью 6 знаков после запятой.

## Структура проекта

- `calc_data.py` — основной Python-скрипт, разбирает аргументы, читает parquet-файл, считает агрегаты и печатает результат.
- `Dockerfile` — сборка контейнера на базе `python:3.14-slim` с зависимостью `polars`.
- `README.md` — описание проекта и команды запуска.

## Как устроено решение

Файл обрабатывается через lazy API библиотеки `polars`, без явной загрузки всего parquet-файла в память. Это важно, потому что по условию файл может быть до 10 ГБ, а лимит памяти контейнера — 4 ГБ.

Из parquet читаются только нужные колонки:

```text
price, quantity, timestamp, is_buyer_maker
```

Для секундной и минутной агрегации программа:

- вычисляет `amount = price * quantity`;
- считает bucket как integer-номер секунды или минуты по `timestamp[ns]`;
- разделяет сделки на buy и sell;
- агрегирует суммы числителя и знаменателя VWAP по bucket'ам;
- считает сумму модулей разницы VWAP для bucket'ов, где есть обе стороны.

Стороны сделок трактуются так:

- `is_buyer_maker=False` — buy;
- `is_buyer_maker=True` — sell.

Bucket учитывается в финальной сумме только если в нем есть обе стороны и у обеих сторон ненулевая сумма `quantity`. Это эквивалентно inner join между buy- и sell-агрегатами.

## Сборка Docker-образа

Из корня проекта:

```bash
docker build -t calc_data .
```

## Запуск через Docker

Пример запуска контейнера:

```bash
docker run --rm -v /path/to/data:/data:ro calc_data /data/input.parquet
```

Где `/path/to/data` — папка на хосте, в которой лежит файл `input.parquet`.
Файл монтируется в контейнер только для чтения через `:ro`.

На Windows PowerShell из папки проекта пример может выглядеть так:

```powershell
docker run --rm -v "${PWD}\data:/data:ro" calc_data /data/input.parquet
```

## Запуск без Docker

Если нужно запустить скрипт напрямую:

```bash
python3 calc_data.py <path_to_parquet>
```

Например:

```bash
python3 calc_data.py /path/to/data/input.parquet
```

Для локального запуска нужны зависимости:

```bash
python3 -m pip install polars==1.41.2
```

## Обработка ошибок

Программа завершится с ненулевым кодом, если:

- не передан путь к parquet-файлу;
- файл не существует или недоступен для чтения;
- parquet-файл нельзя прочитать;
- в файле нет необходимых колонок;
- не установлена зависимость `polars`.

Сообщения об ошибках выводятся в `stderr`.
