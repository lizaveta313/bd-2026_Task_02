# calc_data

Решение задачи обработки parquet-файла с тиковыми сделками Binance по одной торговой паре.

Программа принимает путь к parquet-файлу первым аргументом командной строки и печатает:

```text
VWAP_s=<число>
VWAP_m=<число>
```

Значения выводятся с точностью до 6 знаков после запятой.

## Что считается

Для секундных интервалов считается:

```text
sum_i abs(VWAP_buy_i - VWAP_sell_i)
```

Аналогично считается значение для календарных минут.

Стороны сделок трактуются так:

- `is_buyer_maker=False` — buy;
- `is_buyer_maker=True` — sell.

VWAP считается как:

```text
VWAP = sum(price * quantity) / sum(quantity)
```

В коде числитель VWAP берётся из колонки `quote_qty`, так как для этих данных она соответствует `price * quantity`. Это уменьшает объём вычислений на каждой строке.

Если в секунде или минуте есть сделки только одной стороны, VWAP отсутствующей стороны берётся из ближайшего предыдущего интервала, где такие сделки были. Это реализовано через forward-fill уже после агрегации.

## Как устроено решение

Основная обработка выполняется через lazy API библиотеки `polars` со streaming collect.

Из parquet читаются только нужные колонки:

```text
timestamp, quantity, quote_qty, is_buyer_maker
```

Для ускорения raw-строки группируются только по календарной секунде. Номер минуты вычисляется уже после секундной агрегации как `second // 60`, затем минутные агрегаты считаются из секундных.

Финальная сумма разностей считается через `math.fsum`, чтобы уменьшить ошибку накопления чисел с плавающей точкой.

## Файлы проекта

- `calc_data.py` — основной скрипт расчёта.
- `Dockerfile` — Docker-образ на базе `python:3.14-slim`.
- `.dockerignore` — исключения для Docker build context.
- `.gitignore` — локальные служебные файлы.

## Сборка Docker-образа

```bash
docker build -t calc_data .
```

## Запуск через Docker

```bash
docker run --rm -v /path/to/data:/data:ro calc_data /data/input.parquet
```

## Локальный запуск

```bash
python3 calc_data.py /path/to/input.parquet
```

Для локального запуска нужна зависимость:

```bash
python3 -m pip install polars==1.41.2
```

## Обработка ошибок

Программа завершает работу с ненулевым кодом и пишет сообщение в `stderr`, если:

- не передан путь к parquet-файлу или передано неверное число аргументов;
- указанный файл не существует;
- не установлена библиотека `polars`;
- parquet-файл нельзя прочитать;
- в parquet-файле отсутствуют необходимые колонки;
- во время расчёта возникла ошибка.

При успешном выполнении в `stdout` выводятся только строки `VWAP_s=...` и `VWAP_m=...`.
